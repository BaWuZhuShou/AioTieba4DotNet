# AioTieba4DotNet 标准输出格式

这个文件定义当 skill 生成代码答案时，默认应该采用的中文输出骨架。

## 默认骨架

### 1. 模块判断

先用一句中文点明应该用哪个模块。

示例：

```md
这个需求优先走 `client.Messages`。
```

### 2. 代表方法

列 1-3 个最相关的公开方法，不展开长索引。

示例：

```md
常用公开方法：`GetRepliesAsync()`、`GetAtsAsync()`、`SendMessageAsync(...)`
```

### 3. 最小可运行 C# 示例

这是回答主体。默认要求：

- 单文件片段
- 最少 `using`
- 一个主调用，最多顺带一个紧邻调用
- 1 到 2 行 `Console.WriteLine`
- 默认不用 DI、日志、异常包装、完整项目结构

### 4. 补充提醒

只在必要时补 1 到 2 条：

- 是否需要登录态
- 是否需要吧务权限
- 是否存在并列接口组
- 是否要切换到 `Http` 传输模式

## 紧凑回答模板

// 最小可运行示例
```

- 如果有并列接口，这里说明应选哪一组
````

## 用户明确要“只给代码”时

如果用户明确说“直接给我代码”，就把说明压缩到一两句，其余篇幅给代码块。

## 用户没有指定格式时

默认按本文件输出，不要回到大段解释型答案。
